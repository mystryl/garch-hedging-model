# 滚动回测功能用户指南

## 概述

滚动回测是一种更符合实际操作的风险评估方法。与传统的全样本回测不同，滚动回测会随机选择多个时间窗口进行独立评估，避免 look-ahead bias 和过拟合问题。

## 什么是滚动回测？

### 传统全样本回测的问题

- **Look-ahead bias**：使用未来数据优化过去模型参数
- **过拟合**：在特定数据集上表现良好，但实际应用效果差
- **单一评估**：无法评估不同市场环境下的套保效果

### 滚动回测的优势

- **模拟实际操作**：每个周期独立评估，模拟真实交易场景
- **避免过拟合**：多个窗口独立测试，结果更可靠
- **全面评估**：覆盖不同市场环境（牛市、熊市、震荡市）
- **智能选点**：自动避开交割月份（1月、5月、10月）

## 如何使用

### 1. 启用滚动回测

在 Web 平台的"步骤4.5"中：

1. 勾选"启用滚动回测分析"复选框
2. 配置回测参数（详见下文）
3. 运行模型

### 2. 回测参数配置

#### 回测周期数（n_periods）

- **3个周期**：快速测试，适合数据量较少的情况
- **6个周期（推荐）**：平衡性能和准确性，适合大多数场景
- **9个周期**：详细评估，数据充足时的最佳选择

#### 每个周期天数（window_days）

- **60天（约2个月）**：短期套保，适合交易频繁的策略
- **90天（约3个月，推荐）**：中期套保，平衡效果和稳定性
- **120天（约4个月）**：长期套保，适合稳定的套保需求

#### 最小间隔天数（min_gap_days）

- **120天**：较宽松的限制，数据较少时使用
- **180天（推荐）**：平衡限制，确保周期间独立性
- **240天**：严格限制，最大化周期独立性

### 3. 随机种子模式

#### 固定种子

- **作用**：使用固定数值，确保每次运行选择相同的时间窗口
- **优势**：结果可复现，便于对比不同模型的套保效果
- **适用场景**：模型对比、调试、演示

#### 随机种子

- **作用**：每次运行使用不同的随机种子，选择不同的时间窗口
- **优势**：更全面地评估模型性能，避免偶然性
- **适用场景**：生产环境、最终评估

## 结果解读

### 输出图表

滚动回测会生成 3 张特殊图表：

#### 1. 滚动回测净值曲线（2×3 布局）

- **内容**：每个周期的套保前后净值曲线
- **次坐标轴**：套保比例时序图
- **标题信息**：
  - 周期编号和日期范围
  - 未套保收益率
  - 套保后收益率
  - 方差降低比例

#### 2. 滚动回测回撤曲线（2×3 布局）

- **内容**：每个周期的回撤曲线
- **特殊标记**：最大回撤点（红色倒三角）
- **次坐标轴**：套保比例时序图

#### 3. 周期对比图

- **左上**：各周期收益率对比柱状图
- **右上**：各周期方差降低对比
- **左下**：各周期最大回撤对比（套保后）
- **右下**：汇总统计表格

### 评估指标

#### 平均指标

- **平均收益率（未套保）**：所有周期未套保收益率的平均值
- **平均收益率（套保后）**：所有周期套保后收益率的平均值
- **平均方差降低**：所有周期方差降低的平均值
- **平均最大回撤（套保后）**：所有周期最大回撤的平均值

#### 解释建议

| 方差降低率 | 套保效果评价 |
|-----------|-------------|
| > 90% | 优秀：套保效果显著 |
| 80% - 90% | 良好：套保效果明显 |
| 70% - 80% | 一般：套保有一定效果 |
| < 70% | 较差：需要优化套保策略 |

## 智能选点机制

### 交割月份回避

系统会自动避开以下月份的交割日及其前后30天：
- **1月**：春节前后
- **5月**：劳动节前后
- **10月**：国庆前后

### 日期选择逻辑

1. **筛选有效日期**：排除交割月及其前后30天
2. **确保数据充足**：起始日期后至少有 `window_days` 天的数据
3. **随机选择**：从有效日期中随机选择 `n_periods` 个起始日期
4. **最小间隔**：两个起始日期之间至少间隔 `min_gap_days` 天

## 推荐配置

### 快速测试（数据较少）

```yaml
enable_rolling_backtest: true
n_periods: 3
window_days: 60
min_gap_days: 120
backtest_seed: 42  # 固定种子
```

### 标准配置（推荐）

```yaml
enable_rolling_backtest: true
n_periods: 6
window_days: 90
min_gap_days: 180
backtest_seed: 42  # 固定种子
```

### 详细评估（数据充足）


## 常见问题

### Q1: 为什么结果每次运行都不同？

**A**: 您可能使用了"随机种子"模式。切换到"固定种子"并指定一个数值（如42），结果就会可复现。

### Q2: 滚动回测运行时间较长？

**A**: 这是正常的。滚动回测需要运行多个独立周期，计算量是全样本回测的 N 倍（N = 周期数）。可以减少周期数或每个周期的天数来加速。

### Q3: 如何选择合适的参数？

**A**:
- 数据量少（< 500天）：3个周期，60天/周期
- 数据量中等（500-1000天）：6个周期，90天/周期
- 数据量大（> 1000天）：9个周期，120天/周期

### Q4: 滚动回测和全样本回测哪个更好？

**A**: 滚动回测更符合实际操作，结果更可靠。建议优先使用滚动回测，只有在数据量太少或需要快速测试时才使用全样本回测。

### Q5: 某个周期的结果异常差？

**A**: 这是正常的。滚动回测的目的就是评估不同市场环境下的表现。如果大多数周期效果好，少数周期差，说明模型整体可靠。可以使用固定种子对比不同模型在相同时间窗口下的表现。

## 技术细节

### 实现架构

滚动回测功能基于模块化架构：

- **共享核心**：`lib/shared_backtest/` - 通用回测逻辑
- **模型适配器**：
  - `lib/dcc_garch_analyzer/` - DCC-GARCH 专用
  - `lib/ecm_garch_analyzer/` - ECM-GARCH 专用
  - `basic_garch_analyzer/` - Basic GARCH 专用

### 关键函数

```python
from lib.shared_backtest import run_rolling_backtest

results = run_rolling_backtest(
    data=data,                    # 包含 date, r_s, r_f 列
    hedge_ratios=hedge_ratios,    # 套保比例序列
    n_periods=6,                   # 回测周期数
    window_days=90,                # 每周期天数
    min_gap_days=180,              # 最小间隔
    seed=None,                     # 随机种子（None=随机）
    tax_rate=0.13                  # 税率
)
```

### 返回格式

```python
{
    'period_results': [...],      # 各周期详细结果
    'n_periods': 6,
    'window_days': 90,
    'avg_return_unhedged': 0.05,  # 平均收益率（未套保）
    'avg_return_hedged': 0.02,    # 平均收益率（套保后）
    'avg_variance_reduction': 0.85, # 平均方差降低
    'avg_max_dd_hedged': -0.08    # 平均最大回撤
}
```
