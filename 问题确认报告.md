## 问题1: 滚动窗口协整边界风险 🔴

### 问题描述
当样本量 T ≤ coint_window + 1 时，循环 `for t in range(coint_window + 1, T)` 不会执行，导致：
- `valid_count = 0`
- `beta0_series`, `beta1_series`, `ect` 全为 NaN
- 统计计算 `beta0_series[~np.isnan(beta0_series)].mean()` 触发 `RuntimeWarning` (空数组求均值)

### 问题代码 (model_ecm_garch.py:84-109)
```python
for t in range(coint_window + 1, T):  # 当T=100, coint_window=120时为空
    # ...协整估计
    valid_count += 1

# 统计时变协整参数
print(f"    β0: 均值={beta0_series[~np.isnan(beta0_series)].mean():.4f}")
```

### 验证结果
```python
T = 100, coint_window = 120
range(121, 100) = []  # 空集
beta0_series全为NaN: True
RuntimeWarning: Mean of empty slice
```

### 修正方案
```python
# 添加样本量检查
if T <= coint_window + 1:
    raise ValueError(
        f"样本量不足: T={T}, 需要至少 {coint_window + 2} 个观测值"
    )

# 添加有效样本检查
valid_mask = ~np.isnan(beta0_series)
if valid_mask.sum() == 0:
    raise ValueError(
        f"协整窗口过小或样本量不足,无有效估计点"
    )

# 安全的统计计算
if valid_mask.sum() > 0:
    print(f"β0: 均值={beta0_series[valid_mask].mean():.4f}")
else:
    print("⚠ 警告: 无有效协整估计点,无法计算统计量")
```

## 问题2: ECT对齐混乱 🟡

### 问题描述
- 定义了 `ect_lag = ect[:-1]` 但未实际使用
- 实际使用的是 `ect_aligned = ect[1:-1]`
- 变量命名与实际逻辑脱节，易导致后续维护误解

### 问题代码 (model_ecm_garch.py:129-142)
```python
# 定义了但未使用
ect_lag = ect[:-1]  # 长度T-1

# 实际使用
delta_s_aligned = delta_s[1:]  # 长度T-2
delta_f_aligned = delta_f[1:]  # 长度T-2
ect_aligned = ect[1:-1]  # 长度T-2  <-- 这里!

valid_mask = ~np.isnan(ect_aligned)
ect_lag_valid = ect_aligned[valid_mask]  # 变量名混乱!
```

### 修正方案
```python
# 方案1: 删除未使用的ect_lag，统一使用ect_aligned
# 方案2: 重新命名，使逻辑清晰

# 推荐方案1:
# 对齐说明: ECM使用滞后一期的ect来解释当期的价格差分
# ΔS(t) = α + h·ΔF(t) + γ·ect(t-1) + ε(t)
#
# 由于:
# - ΔS(t) 对应 delta_s[t-1] (从t=1开始,即第2个价格差分)
# - ect(t-1) 对应 ect[t-2] (从t=1开始,即第2个ect)
#
# 因此需要:
delta_s_aligned = delta_s[1:]      # ΔS(2), ΔS(3), ..., ΔS(T)
delta_f_aligned = delta_f[1:]      # ΔF(2), ΔF(3), ..., ΔF(T)
ect_lagged = ect[1:-1]               # ect(1), ect(2), ..., ect(T-1)

valid_mask = ~np.isnan(ect_lagged)
delta_s_valid = delta_s_aligned[valid_mask]
delta_f_valid = delta_f_aligned[valid_mask]
ect_lagged_valid = ect_lagged[valid_mask]

# 删除未使用的变量
# ect_lag = ect[:-1]  # <-- 删除这行
```

### 时序对应关系澄清
```
时间索引:   0    1    2    3   ...  T-2  T-1   T
─────────────────────────────────────────────────
spot:      S0   S1   S2   S3   ... ST-2 ST-1  ST
futures:   F0   F1   F2   F3   ... FT-2 FT-1  FT
ΔS:        -   ΔS1  ΔS2  ΔS3  ... ΔST-2 ΔST-1  -
ΔF:        -   ΔF1  ΔF2  ΔF3  ... ΔFT-2 ΔFT-1  -
ect:       ect0 ect1 ect2 ect3 ... ectT-2 ectT-1 ectT

ECM回归:
  被解释变量: ΔS(t) for t=2,...,T  →  delta_s[1:]
  解释变量1:  ΔF(t) for t=2,...,T  →  delta_f[1:]
  解释变量2:  ect(t-1) for t=2,...,T →  ect[1:-1]
```

## 问题3: 协整检验趋势项过度拟合 🟠

### 问题描述
- 当前使用 `coint(spot, futures, trend='ct', ...)`
- `trend='ct'` 包含 **常数项 + 时间趋势项**
- 现货-期货的长期均衡关系通常是 **线性且无时间趋势**：`S = β0 + β1*F`
- 引入时间趋势项会导致过度拟合，可能误判协整关系

### 理论依据
**Engle-Granger协整检验的标准做法**:
- `trend='c'`: 仅包含常数项（最常用）
- `trend='ct'`: 包含常数项和趋势项（仅当有明确趋势时使用）
- `trend='nc'`: 无常数项，无趋势项（很少使用）

对于现货-期货价格：
- 两个价格序列通常都包含单位根（非平稳）
- 它们的线性组合应该是平稳的（无趋势）
- **标准做法: `trend='c'`**

### 问题代码 (model_ecm_garch.py:60)
```python
score, pvalue, _ = coint(spot, futures, trend='ct', autolag='BIC')
```

### 修正方案
```python
# 修改为trend='c' (仅常数项)
score, pvalue, _ = coint(spot, futures, trend='c', autolag='BIC')

# 如果需要验证，可以同时尝试两种趋势
for trend_type in ['c', 'ct']:
    score, pvalue, _ = coint(spot, futures, trend=trend_type, autolag='BIC')
    print(f"  trend='{trend_type}': 统计量={score:.4f}, p值={pvalue:.6f}")
    if pvalue < 0.05:
        print(f"    ✓ 在trend='{trend_type}'下存在协整关系")
```

### 影响
- **过度拟合**: 引入不必要的参数降低检验势
- **误判风险**: 可能在不存在协整时拒绝原假设
- **理论不一致**: 与金融理论不符

## 问题4: 缺乏ARCH效应检验 🟠

### 问题描述
- GARCH模型的前提是残差存在 **ARCH效应**（异方差性）
- 当前代码直接对ECM残差拟合GARCH，未进行ARCH-LM检验
- 模型设定缺乏统计依据，可能存在设定误差

### ARCH效应检验的重要性
**GARCH模型适用条件**:
1. 残差序列存在自相关
2. 残差平方序列存在自相关（即存在ARCH效应）
3. 通过 **ARCH-LM检验** 确认异方差性

如果不满足这些条件：
- GARCH模型可能不适用
- 应考虑使用其他模型（如OLS、EWMA等）

### 修正方案
```python
from statsmodels.stats.diagnostic import het_arch

# 步骤1: 对ECM残差进行ARCH-LM检验
print("\n[步骤3.5] ARCH效应检验...")

# 检验滞后阶数
lags = [5, 10, 20]  # 常用的滞后阶数
arch_results = []

for lag in lags:
    lm_stat, p_value, _ = het_arch(ecm_residuals, nlags=lag)
    arch_results.append({
        'lag': lag,
        'lm_statistic': lm_stat,
        'p_value': p_value,
        'has_arch': p_value < 0.05
    })
    print(f"  ARCH-LM检验(lag={lag}):")
    print(f"    LM统计量: {lm_stat:.4f}")
    print(f"    p值: {p_value:.6f}")
    print(f"    结论: {'✓ 存在ARCH效应' if p_value < 0.05 else '✗ 不存在ARCH效应'}")

# 步骤2: 根据检验结果决定是否使用GARCH
if any(r['has_arch'] for r in arch_results):
    print("\n  ✓ 残差存在ARCH效应，适合使用GARCH模型")
    # 继续拟合GARCH...
else:
    print("\n  ⚠ 残差不存在ARCH效应")
    print("  建议: 考虑使用简单OLS或滚动标准差模型")
    # 仍然可以拟合GARCH，但需要注意解释
```

### 标准的模型设定流程
```
1. 拟合ECM方程 → 得到残差 ε(t)
2. 残差诊断检验:
   a. 自相关检验 (Ljung-Box)
   b. ARCH效应检验 (ARCH-LM)
   c. 正态性检验 (Jarque-Bera)
3. 如果存在ARCH效应 → 拟合GARCH
4. 如果不存在ARCH效应 → 使用OLS或简单方法
```
